JSX
It seems like React components are returning HTML markup. However, this is not the case. The layout of React components is mostly written using JSX[LINK_HERE]. Although JSX looks like HTML, we are dealing with a way to write JavaScript. Under the hood, JSX returned by React components is compiled into JavaScript.
After compiling, our application looks like this:
[CODE_SNIPPET_HERE]
The compilation is handled by Babel[LINK_HERE]. Projects created with Vite are configured to compile automatically. We will learn more about this topic in part 7[LINK_HERE] of this course.
It is also possible to write React as "pure JavaScript" without using JSX. Although, nobody with a sound mind would do so.
In practice, JSX is much like HTML with the distinction that with JSX you can easily embed dynamic content by writing appropriate JavaScript within curly braces. The idea of JSX is quite similar to many templating languages, such as Thymeleaf used along with Java Spring, which are used on servers.
JSX is "XML[LINK_HERE]-like", which means that every tag needs to be closed. For example, a newline is an empty element, which in HTML can be written as follows:

Writing components with React is easy, and by combining components, an even more complex application can be kept fairly maintainable. Indeed, a core philosophy of React is composing applications from many specialized reusable components.

Nevertheless, as we will learn in part 6[LINK_HERE], there are situations where the component App is not exactly the root, but is wrapped within an appropriate utility component.
